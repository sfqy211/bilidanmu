# 页面设计

> 管理面板（4 个子页面）、弹幕页面、系统托盘的 UI 规范与交互细节。

---

## 一、页面总览

两类页面：**管理面板**（侧边栏 + Tab 内容区）、**弹幕页面**（独立全屏）。

│                         BiliDanmu                               │
├─────────────┬───────────────────────────────────────────────────┤
│             │                                                   │
│  管理面板    │  弹幕页面（独立全屏布局）                           │

│   通过 Tab  │                                                   │

│             │  │          实时弹幕流 + 发送栏                  │ │

## 二、路由结构

```
/                           → 重定向到 /rooms
/rooms                      → 管理面板 - 直播间（Tab: 已添加 / 搜索）
/accounts                   → 管理面板 - 账号（Tab: 登录 / 账号）
/ai                         → 管理面板 - AI 接入（Tab: 添加模型 / 已保存）
/settings                   → 管理面板 - 设置（Tab: 弹幕发送 / 弹幕接收 / 外观 / 通知）
/room/:roomId               → 弹幕页面（独立全屏布局）
```

```tsx
<Routes>
  <Route element={<AppLayout />}>
    {/* 侧边栏(w-16) + 内容区 */}
    <Route path="/rooms" element={<RoomPage />} />
    <Route path="/accounts" element={<AccountPage />} />
    <Route path="/ai" element={<AIPage />} />
    <Route path="/settings" element={<SettingsPage />} />
  </Route>
  <Route path="/room/:roomId" element={<DanmakuPage />} /> {/* 独立布局 */}
</Routes>
```

## 三、管理面板

**整体布局：** 左侧 64px 图标侧边栏 + 右侧内容区

```
┌──────────────────────────────────────────────────────┐
│  BiliDanmu (BD)                          ─  □  ✕    │
├────────────┬─────────────────────────────────────────┤
│            │                                         │
│  📺 直播间 │        页面标题 + 描述                    │
│  👤 账号   │        ──────────────────────            │
│  🤖 AI    │        [Tab A] [Tab B]                   │
│  ⚙️ 设置  │        ┌─────────────────────────────┐   │
│            │        │                             │   │
│────────────│        │     Tab 内容区               │   │
│ 版本号     │        │                             │   │
└────────────┴─────────────────────────────────────────┘
  侧边栏 w-16     内容区自适应
  (图标 + 左侧     (页面标题 + PageTabs + 内容)
   粉色竖条指示)
```

**侧边栏规范：**

- 宽度 `w-16`（64px），顶部 "BD" 品牌标识，仅显示图标，hover 显示文字 tooltip
- 当前选中项：左侧 3px 粉色竖条 + 背景色变化
- 底部：版本号
- 4 个导航项：直播间、账号、AI 接入、设置

**页面通用模式：**

所有管理页面共享相同结构：

1. 页面标题（`text-2xl font-semibold`）+ 灰色描述文案
2. 右上角操作按钮 / 状态提示（error/success）
3. `PageTabs`（Radix Tabs）切换内容区域
4. 内容区使用统一的边框卡片样式（`border-slate-300 bg-white` / 暗色 `border-white/[0.06] bg-[#12141e]`）

**交互细节：**

5. 删除当前房间时自动保存 `selections.save({ currentRoomId: null })`

### 3.2 子页面二：账号（AccountPage）

**Tab 结构：** `登录` | `账号`

```
┌─────────────────────────────────────────────────────────┐
│  账号                                                    │
│  扫码或 Cookie 登录，管理多个账号。                       │
├────────┬────────────────────────────────────────────────┤
│ 登录   │ 账号                                           │
├────────┴────────────────────────────────────────────────┤
│                                                         │
│  ──── 登录 Tab（左右双栏） ────                          │
│  ┌─────────────────────┐ ┌─────────────────────────┐   │
│  │ 扫码登录             │ │ Cookie 登录             │   │
│  │ ┌─────────────┐     │ │ ┌───────────────────┐  │   │
│  │ │  QR 二维码   │     │ │ │  粘贴完整 Cookie  │  │   │
│  │ │  (本地生成)  │     │ │ │  字符串...         │  │   │
│  │ └─────────────┘     │ │ └───────────────────┘  │   │
│  │ [生成二维码]         │ │ [使用 Cookie 登录]      │   │
│  │ 请使用 App 扫码...   │ │ 需包含 SESSDATA         │   │
│  └─────────────────────┘ └─────────────────────────┘   │
│                                                         │
│  ──── 账号 Tab ────                                     │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 头像  用户A  [当前]  UID: 123456                  │   │
│  │                    [切换] [移除]                  │   │
│  ├─────────────────────────────────────────────────┤   │
│  │ 头像  用户B             UID: 789012              │   │
│  │                    [切换] [移除]                  │   │
│  └─────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 快捷操作                              [退出当前] │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

**交互细节：**

1. **扫码登录**：点击「生成二维码」→ `qrcode` 库本地渲染 → 每 3s 轮询 `poll_qr` → 过期自动刷新 → 成功后写入 auth-store
2. **Cookie 登录**：粘贴完整 Cookie 字符串 → textarea 输入 → 提示需包含 `SESSDATA` 与 `bili_jct`
3. **多账号列表**：头像、用户名、UID、当前标记；切换/移除按钮
4. **移除账号**：后端自动激活下一个账号，返回新活跃 ID 前端同步
5. **退出登录**：返回剩余账号列表，前端用列表更新而非清空
6. 托盘菜单同步显示所有账号，CheckMenuItem 切换

**交互细节：**

1. **添加/编辑共享表单**：同一 Tab，编辑时标题切换为「编辑模型」+ 显示「取消编辑」按钮

5. **已保存列表**：2 列网格卡片，当前模型粉色高亮 + "当前" 标签；操作：编辑、设为当前、删除
6. ID 使用时间戳 + 随机 hex 防冲突，持久化至 SQLite

## 四、弹幕页面（DanmakuPage）

**全屏独立布局**，无侧边栏，进入房间后自动连接弹幕流。三栏布局：礼物栏（上）、弹幕+SC 栏（下）、入场信息（底部单行）。

```
┌──────────────────────────────────────────────────────────────────┐
│  音频控制栏（播放/暂停 + 音量 + 静音）                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│   礼物栏                                                         │
│   🎁 用户D 送出 小电视x1                                         │
│   🎁 用户F 送出 醒目留言x1                                       │
│                                                     [回到底部 ↓] │
│                                                                  │
├─ ─ ─ ─ ─ ─ ─ ─ 可拖动分割栏 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┤
│                                                                  │
│   弹幕+SC 栏                                                     │
│   [勋章] 用户A: 哈哈哈                                           │
│   [勋章] 用户B: [emote_img]                                      │
│   💬 用户C 大表情                                                │
│   ┌──────────────────────────────────────────────────────────┐   │
│   │ 💰 SC 卡片：头像 + 用户名 + ¥30                          │   │
│   │ 底部：SC 消息内容                                        │   │
│   └──────────────────────────────────────────────────────────┘   │
│                                                     [回到底部 ↓] │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│  ↪ 用户E 进入了直播间（仅显示最新一条）                           │
├──────────────────────────────────────────────────────────────────┤
│  [🔁] [😊] [___输入弹幕内容___] [➤]                              │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ 表情选择器 / 自动发送面板（浮动在输入栏上方）               │   │
│  └──────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

**三栏消息分流：**

| 消息类型         | 分配栏位   | 渲染方式                             |
| ---------------- | ---------- | ------------------------------------ |
| 礼物 (`gift`)    | 礼物栏     | `🎁 用户 送出 礼物名`，amber 金色    |
| 弹幕 (`danmaku`) | 弹幕+SC 栏 | `[勋章] 用户名: 消息文本`            |
| SC (`superChat`) | 弹幕+SC 栏 | SuperChatCard 两段式卡片（真实颜色） |
| 系统 (`system`)  | 弹幕+SC 栏 | 系统消息样式                         |
| 入场 (`entry`)   | 底部单行   | 仅显示最后一条，固定高度             |

**分割栏：** 可拖动调整礼物栏/弹幕栏高度比例，默认 35:65。拖到 <5% 只显示弹幕，拖到 >95% 只显示礼物。比例持久化到 localStorage。高度 6px，hover 时显示颜色提示。

**弹幕输入栏：**

- 左侧自动发送按钮（🔁）→ 弹出 AutoSendPanel 浮动面板
- 表情按钮（😊）→ 弹出 EmoticonPickerPanel 浮动面板
- 中间单行输入框，Enter 发送，最多 20 字
- 右侧发送按钮（➤），发送中 disabled

**表情选择器（EmoticonPickerPanel）：**

- 标题 + 关闭按钮
- 表情包横向滚动 Tab（封面图标 + 包名 tooltip），`makePkgKey(pkgId-pkgType)` 复合 key 切换
- 下方表情网格（3-6 列响应式），不可用表情灰显 + cursor-not-allowed
- 所有图片通过 `ProxiedImage` 组件自动代理 B 站 CDN

**自动发送面板（AutoSendPanel）：**

- 四个 Tab：文字 / 表情 / 收藏夹 / 点赞
- 文字 Tab：多行 textarea（每行一条），收藏夹可填充到文字
- 表情 Tab：表情选择器 + 已选预览
- 收藏夹 Tab：多弹幕组管理（添加/删除/编辑）
- 点赞 Tab：目标总数/批次/间隔参数 + 启停
- 共享控件：间隔（秒）+ 时间限制（分钟）+ 启动/停止按钮
- 运行状态：已发送 N 条 / 当前第 M 条 / 停止原因
- 切换房间或组件卸载时自动停止（`isRunningRef` 条件检查）

**弹幕流渲染规则：**

| 消息类型       | 渲染方式                             | 样式                |
| -------------- | ------------------------------------ | ------------------- |
| 普通弹幕       | `[勋章] 用户名: 消息文本`            | slate 文字          |
| 含文字表情弹幕 | `[emote_key]` 替换为 inline `<img>`  | InlineEmotText 混排 |
| 大表情弹幕     | 仅表情图片（official\_ 限制 56px）   | 居中 inline         |
| 醒目留言 (SC)  | SuperChatCard 两段式卡片（真实颜色） | 独立卡片            |
| 礼物消息       | `🎁 用户 送出 礼物名`                | amber 金色          |
| 进场消息       | `↪ 用户 进入了直播间`                | 灰色小字            |

**图片代理：**

所有 B 站 CDN 图片（`*.hdslb.com`）通过 `ProxiedImage` → `useProxyImage` → `proxy_image` IPC 命令代理加载，绕过 Referer 防盗链。代理后的 base64 data URL 缓存在前端 LRU（200 条），代理失败时 fallback 到原始 URL。

**音频控制栏：** 底部输入栏上方，包含播放/暂停按钮、音量滑块、静音按钮。右侧显示开播时长（`Clock` 图标 + `XhXm` 格式，每 5 分钟刷新 `get_live_time`，每分钟更新显示）、在线人数、点赞总数。点击播放自动调用 `get_audio_stream_url` 获取代理 URL，mpegts.js 连接本地代理播放 FLV 音频流。切房时先 clear 再 destroy，自动重连 3 次（4s 间隔）。

**字幕叠加层（SubtitleOverlay）：** 覆盖在弹幕流区域底部，半透明黑底白字，显示 STT 实时识别结果。非最终结果时显示为"正在说话"状态，最终结果后淡出 800ms，5 秒后消失。通过 `useSttTranscript` hook 管理延迟缓冲和按需 RAF 循环。
